先说实在话：我的实验室能托管东西、能部署东西，但没有一个地方能*按计划跑一段逻辑，然后拿结果做点事*。这周我把这个缺口补上了——我立起了 [Dagster](/data/dagster)，一个数据/DAG 编排器，作为一个货真价实的新服务类别："数据 / 编排"。而我拿它做的第一件事，就是把它掉转过来指向集群自身：一条每日流水线，读我自己的 GPU 指标，写出一段关于编队状态的英文摘要。这是自那台会部署自己的 Git 服务器之后，我造过的最自我指涉的东西。

<!-- truncate -->

## 为什么要开一个全新的类别

我往实验室里加的大多数东西都能塞进现成的架子：又一个媒体应用、又一个平台服务、又一个模型服务器。Dagster 哪个都塞不进去，而这恰恰是它有意思的地方。实验室里其他一切都是*一个会运行的东西*。Dagster 是*那个拿着我的代码去跑其他东西的东西*。它是不同的一层——覆盖在平台之上的可编程黏合层——所以它拿到了自己的类别，而不是被硬塞进"平台服务"里。

管道部分几乎是无聊的，而这是最好的那种无聊。Dagster 是官方 Helm chart，通过 kustomize 展开（和 open-webui 同一套路），由一个 Argo CD 应用部署，元数据存储是一个固定在 a2 的朴素 `postgres:16-alpine`。唯一那处漂亮的架构点睛是 **K8sRunLauncher**：每次流水线运行都作为它自己的一个 Kubernetes Job 执行。没有长期存活的 worker 池——Dagster 为每次运行向集群要一个全新的 Pod，你就在 `kubectl get jobs` 里看着它们出现又消失。编排器把执行直接下放给了它本就坐落其上的调度器。这是对 k8s 原生编排一次可爱又清晰的演示。

而流水线*代码*根本不需要一套新的部署故事。它住在自己的 Forgejo 仓库里，由构建其他一切的同一个 Forgejo Actions 运行器构建成 `harbor.lan/apps/dagster-pipelines:<sha>`，再通过碰一下镜像 tag、由 Argo 滚动出去来晋级。Dagster 就这么插进了本来就在那儿的 [CI 环](/gitops/ci-loops)。这就是先把机器造好的回报：你加的第五样东西，成本只是第一样的一个零头。

## 旗舰：一条描述自己硬件的流水线

第一条真正的流水线叫"GPU 摘要"，它是把整个实验室浓缩进一次作业的一趟巡礼：

1. 它查询我自己的 Prometheus，取 dcgm-exporter 的指标——每块 GPU 的利用率、显存、功耗、温度——外加自定义的 vram-reporter 取显存最饥渴的那些 Pod。
2. 它把这些数字交给我自己的 [LiteLLM 网关](/ai/litellm)，要一段自然语言的"GPU 编队状态"摘要（网关挂了还有模板兜底）。
3. 它每天跑。

模型通过一个环境变量配置，默认是本地的 `nemotron-omni`，于是整件事——数据、推理、编排——都发生在我自己的硬件上，环里没有一点云。Dagster 从 Prometheus 读，用 LiteLLM 思考，经由 Forgejo 和 Harbor 发布。那是被一个新系统串起来的四个现有系统，而这正是把它叫作黏合层的全部意义。

## 然后 PII 卫兵给我的 GPU Pod 改了名

让我笑出来的时刻在这里。第一份摘要回来时，兴高采烈地报告了一个名叫 `DRIVERS_LICENSE_1` 的 Pod 的 GPU 利用率。

没有任何东西坏了。那是 [Rampart](/ai/rampart)，我的 PII 脱敏卫兵，正在干它分内的活。每一次发往 LiteLLM 网关的 prompt 都会过一遍 Rampart，而一个哈希过的 Pod 名显然看起来足够像一个身份证号，脱敏器就在模型看到它之前把它换成了一个 `DRIVERS_LICENSE` 占位符。我的安全层伸出手，把我自己基础设施的主机名给打了码。

我爱这个失败，因为它是好的那种。它证明了脱敏器是真的在链路里——不是一份*声称*保护云调用的配置，而是一个真的会重写每一条 prompt 的过滤器，连我没想到需要保护的那些也一并重写了。过度脱敏再修文字，是安全的那个失败方向。修法显而易见、也在清单上：在 Pod 名撞上网关之前先规范化。但我对那份为我的 RTX 4090 操心驾照状态的摘要，有一种奇怪的喜爱。

## 这解锁了什么

Dagster 现在是"按计划面向平台做某件 X"的所在。GPU 摘要是第一个租户；接下来那些显而易见的自己都能写出来——每晚的备份验证报告、从 Forgejo 提交拼装出的每周更新日志、从 LiteLLM 数据库汇总的成本/用量。这一层现在存在了，而且它能看见家里其他一切。

有一个我想留在记录里的诚实告诫：Dagster OSS 没有内建鉴权，所以它纯靠 LAN 和 tailnet 把关。对一个躲在默认拒绝的 tailnet 后面的单人操作者来说，这没问题，也正因此，在没有鉴权代理挡在前面之前它不离开 tailnet。就绪不等于可以安全暴露——这个教训，这座实验室换着戏服反复教我。
