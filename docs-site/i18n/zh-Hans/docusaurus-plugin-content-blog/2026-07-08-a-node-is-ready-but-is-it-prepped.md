先说实在话：给 k3s 加一个节点简单得有点骗人。你跑一个脚本，节点打印出 `Ready`，所有直觉都告诉你搞定了。你没搞定。`Ready` 意味着 kubelet 在和 API server 通话——仅此而已。它对这台机器是否已经被*准备好*成为集群里一个可靠成员只字不提，而在我最新的这个节点上，那些准备工作几乎一件都没真的做过。它是靠运气 `Ready` 的。

<!-- truncate -->

这周，一台老旧的 ThinkPad T430——四核、8 GB 内存、没有 GPU、在自己的子网上——作为[第六个节点](/hardware/nodes)加入了集群。它是编队里最弱的机器，而这正是重点：它吸收一些小型的纯 CPU 服务，好让 RTX 4090 机器专注于 GPU 的活。和我另一台笔记本节点一样，它被刻意排除在存储成员之外，也不承担 GPU 角色。一台孱弱、仅靠 WiFi 的笔记本很适合跑一个扫描器 sidecar，却很不适合存放复制数据。

## 陷阱：加入不等于准备好

差点坑到我的就是这件事。这个节点已经*加入*了——它在 `kubectl get nodes` 里显示为 `Ready`，Pod 在往上调度，一切看着都是绿的。但加入集群只需要一个 token。真正让一台笔记本节点保持*健康*的东西全是另外的主机准备工作，而在 T430 上，这些被悄悄跳过了：

- **屏蔽睡眠和挂起，忽略合盖。** 笔记本的默认反射就是去睡觉。一个睡着的节点就是 `NotReady` 节点。
- **关闭 WiFi 省电。** 就是这一项设置，能防止一台 WiFi 笔记本节点在轻负载下抖进抖出 `NotReady`。开着它，节点看起来就像闹鬼。
- **调高 inotify sysctl。** 密集的、监视文件的 Pod 会耗尽默认上限，然后带着一堆和 Pod 本身毫无关系的晦涩错误 crashloop。
- **信任局域网 CA 并固定 `harbor.lan`。** 没有这些，节点没法从我的镜像仓库拉取，也没法通过 TLS 和 `.lan` 服务通话。
- **把它加入 tailnet。** 这样我就能像对待其余机器一样远程访问它。

这些没有一样是 `Ready` 所暗示的。我反复重新学到的教训：绿色的节点状态告诉你 kubelet 活着，而这就是它告诉你的*全部*。去核实准备工作真的做过——别信状态。"它加入了"和"它可以干活了"是两个不同的论断，而只有其中一个会出现在 `kubectl` 里。

## 三处手动的可观测性动作

上线的另一半，是让这个节点*可见*。我的可观测性大部分是 DaemonSet——node-exporter、把日志运往 Loki 的 promtail、netdata、Scrutiny 磁盘健康、node-feature-discovery——所以节点一加入，它们就自动落上去了。这部分确实零成本。

但有三件事顽固地是手动的，每个子系统一件，而且每一件忘了都会无声地失败：

1. **Prometheus 抓取目标。** 我的监控是[刻意手写的](/observability/prometheus-grafana)——纯 `static_configs`，没有 Operator，没有 ServiceMonitor，因为我看重能在一个文件里读完整套配置。这份可读性的代价是：新节点的 node-exporter 立刻就跑，但在我手动把目标加进去之前，Prometheus 不会去抓它。Operator 会自己发现它；我选了可读的文件，也就选了随之而来的手动编辑。
2. **fleet 仪表盘的节点标签映射。** 我的 Grafana 仪表盘里，温度标签有一份烤死的 IP-到-节点名映射。在我把新节点的 IP 加进去之前，它那一行是空白渲染的——数据在流，却无处落脚。
3. **重启 Dozzle。** 这个是真正值得[记下来](/observability/logs)的怪癖。[Dozzle](/observability/logs) 在 Kubernetes 模式下作为单个 Pod 运行，*只在启动时枚举一次*集群的节点。之后才加入的节点对它就是隐形的——没有报错，没有警告，就是缺了。一次 `kubectl rollout restart`，它就出现了。失败方式是无声的，这恰恰是为什么它容易让你搭进去一个下午。

## "给节点上线"的形状

于是真正的定义，我以后要记住的这个：**给节点上线 = 加入 + 准备 + 三处手动的可观测性动作。** 加入是简单的那 10%。准备是让它能持续在线的东西。而那三处动作，是让它出现在你真正会去看的仪表盘上——在凌晨 5:40 出事时你会盯着的那些。

好消息是趋势的方向：DaemonSet 已经自动扛下了重活，而手动的那几块都很小、都在 git 里、也都是做成一个正经上线脚本的候选。T430 现在被完整地欢迎进来了——屏蔽了、固定了、信任了、抓取了、画进图了、也在 tailing 了。但它又一次教会我：不要相信一个我没挣来的绿色对勾。
