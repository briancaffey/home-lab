# Phoenix — Arize Phoenix (LLM observability)

Self-hosted LLM trace explorer + evals. Receives OTLP traces (from LiteLLM, apps,
the agent) and lets you explore spans, prompts, and run evaluations.

Deployed from the **official OCI Helm chart** as a **Helm release** (not kustomize):
the chart is OCI-published, autogenerates its own in-cluster Secret, and bundles a
Postgres subchart — Helm handles all three natively. This replaced the earlier
hand-rolled Deployment that used SQLite on a local-path PVC.

## Layout
- **App:** `phoenix` Deployment, pinned to **a3**, ClusterIP `phoenix-svc`.
- **DB:** bundled Postgres StatefulSet `phoenix-postgresql`, pinned to **a3**,
  10Gi local-path PVC. 30-day trace retention (bounds DB growth).
- **Service:** `phoenix-svc` — `6006` (UI / HTTP-OTLP), `4317` (gRPC-OTLP).
- **Auth:** OFF (LAN/tailnet-only). To enable, see `values.yaml`.

## Access
- LAN: https://phoenix.lan/ (Traefik, `phoenix-tls` mkcert secret, Homepage-discovered)
- Tailnet: https://phoenix.\<tailnet\>.ts.net/ (Tailscale operator)

## Deploy / upgrade
```bash
# 1. TLS secret for phoenix.lan (one-time / when rotating certs)
bash scripts/lan-certs.sh

# 2. The release (Postgres-backed)
helm upgrade --install phoenix \
  oci://registry-1.docker.io/arizephoenix/phoenix-helm --version 9.0.11 \
  -n observability -f observability/phoenix/values.yaml

# 3. Ingresses (raw manifests)
kubectl apply -f observability/phoenix/ingress-lan.yaml
kubectl apply -k clusters/home/tailscale     # includes ingress-phoenix.yaml
```

## Notes
- **LiteLLM** exports to `phoenix-svc.observability.svc.cluster.local:4317` (grpc).
- The chart's `phoenix-secret` is generated in-cluster (never committed). With auth
  off, the autogenerated `PHOENIX_SECRET` is functionally irrelevant; the Postgres
  password is static, so the DB connection is stable across `helm upgrade`s.
- Migrating from the old hand-rolled manifest is destructive (the old SQLite PVC
  `phoenix-data` is deleted). Trace history does not carry over.
